## Reporting a Vulnerability

If you discover a security vulnerability, please **do not create a public issue**. Instead, report it privately to the maintainers:

- **Email:** moizyopensource@gmail.com (replace with your contact email)
- Subject: `[SECURITY] <Short Description>`

Please include the following information:

- Description of the vulnerability
- Steps to reproduce
- Impact assessment
- Any suggested fixes or patches (optional)
- Your contact information for follow-up

The maintainers will respond within **48 hours**.

---

## Supported Security Practices

- We encourage contributors to follow **best security practices** in code:
  - Validate user input to prevent injection attacks.
  - Avoid storing sensitive information in code or repos.
  - Follow proper authentication & authorization practices.
- Regularly update dependencies to fix known vulnerabilities.

---

## Security Updates

- Once a vulnerability is confirmed, maintainers will:
  1. Privately fix the issue or merge a PR.
  2. Release a **patched version**.
  3. Publicly disclose the security fix responsibly after release.

- All contributors are encouraged to **review pull requests** for security risks.
